## How to Fix

1. **Find your Supabase Site URL setting**
   - Supabase Dashboard â†’ Authentication â†’ URL Configuration
   
2. **Change it from:**
   ```
   http://localhost:3000
   ```
   
3. **To your deployed URL:**
   ```
   https://your-deployed-site.vercel.app
   ```

4. **Keep both in Redirect URLs** (for development):
   ```
   https://your-deployed-site.vercel.app/**
   http://localhost:3000/**
   ```

5. **Save and test!**

## Why This Happens

When you first set up Supabase, you were developing locally, so you set:
```
Site URL = http://localhost:3000
```

This worked great for development! But when you deployed to production, you forgot to update this setting. So now:

- âœ… Local development: Works fine (redirects to localhost)
- âŒ Production: Broken (still redirects to localhost!)

The fix is simple: Update the Site URL to your production domain.
